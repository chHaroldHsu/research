### 三、論文選題的契合度判斷框架

當思考一個候選題目時，可用以下三維度檢視：

#### 維度 A：與楊老師方法論的契合度
- [ ] 是否包含可視化／視覺化呈現？
- [ ] 是否需要演算法設計或改良？
- [ ] 是否處理多媒體資料（影像、軌跡、空間資料）？
- [ ] 是否能做量化評估？

#### 維度 B：與智慧製造系所的契合度
- [ ] 能否歸入「工業數據科學 / 智慧製造與營運 / 機器人工程與自動化技術」其中一類？
- [ ] 是否能對應到製造／物流／營運場域的實務問題？
- [ ] 即使取材自其他場域（智慧城市、交通），是否能做學術遷移論述？

#### 維度 C：碩論可行性
- [ ] 資料是否取得得到（開放資料、模擬、自行蒐集）？
- [ ] 在 1～1.5 年內能否完成「baseline + 改良 + 評估」？
- [ ] 是否有清楚的 gap，能說明「別人還沒做好什麼」？

### 四、個人研究興趣與題目發想

#### 4.1 廣義興趣領域

目前對以下幾個方向有持續的關注與自發研究動機：

- **視覺化與動態模擬**：喜歡以動態、互動的方式呈現系統運作過程，接近 Nature of Code 的 creative coding 取向
- **城市與交通系統**：對 YouBike 站點供需調度、車流預測、群眾行為模擬等問題有具體的觀察與思考
- **演算法驅動的問題解決**：對透過設計或改良演算法來解決真實場域問題有興趣，特別是排程、路徑規劃、狀態推斷等類型

#### 4.2 視覺化興趣的學術定位釐清

學術視覺化論文貢獻通常落在四種類型：技術型、設計型、系統型、評估型。

**個人定位**：視覺化在論文中扮演**支援性角色**，幫助呈現演算法結果與增加系統可用性。論文的**實質貢獻**應落在演算法設計、問題解決、或數據分析方法上，而非「做了一個漂亮的視覺化系統」。借用 Nature of Code 風格是資產，但學術貢獻必須建立在真實資料、量化評估、與既有方法的對話之上。

#### 4.3 場域興趣遷移至智慧製造

城市／交通場域的興趣可遷移至智慧製造，作為題目合法性論述的依據：

1. **YouBike 站點供需失衡** → 產線 WIP buffer 失衡、跨站料件調度
2. **群眾協作調度（使用者繞路）** → 柔性人力派遣、跨站作業員調度
3. **紅綠燈秒數反推（從觀察反推系統狀態）** → 老舊設備未連網時，從產出節奏反推設備週期與瓶頸
4. **車流路徑規劃** → AGV／AMR 多車路徑規劃、廠內物流
5. **即時交通預測** → 即時產能預測、瓶頸預警
6. **App 遊戲化激勵** → 作業員績效視覺化、產線改善提案系統
